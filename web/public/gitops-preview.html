Declarative GitOps Restart (Argo CD compatible)

Instead of using kubectl rollout restart (imperative), this patches the kubectl.kubernetes.io/restartedAt annotation on the deployment template. The annotation change is declarative and can be committed to Git so Argo CD picks it up automatically.

Declarative Patch (commit to Git)

Add this to your deployment manifest in Git. Argo CD will detect the annotation change and trigger a rolling restart automatically.

spec:
  template:
    metadata:
      annotations:
        kubectl.kubernetes.io/restartedAt: "<timestamp>"

Click "Apply Declarative Restart" to generate a real timestamp, or replace <timestamp> with the current UTC time.

How it works with Argo CD

  1. 1 Update the restartedAt annotation in your deployment YAML in Git.
  2. 2 Commit and push the change. Argo CD detects the diff and marks the app as OutOfSync.
  3. 3 Argo CD syncs the application, applying the annotation change which triggers Kubernetes to perform a rolling restart.
  4. 4 The restart is fully auditable via Git history — no imperative commands needed.
